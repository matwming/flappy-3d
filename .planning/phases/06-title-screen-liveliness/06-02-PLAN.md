### Changes to TitleScreen.tsx

1. Add `useRef` to the Preact hooks import (already has `useEffect`):
```typescript
import { useEffect, useRef } from 'preact/hooks'
```

2. Add `gsap` import at the top:

```typescript
import { gsap } from 'gsap'
```

3. Inside the `TitleScreen` component function, add the `hasAnimated` ref and
   the logo animation `useEffect`:

```typescript
const hasAnimated = useRef(false)

useEffect(() => {
  // One-shot: skip if already animated this session (e.g. back-to-title from gameOver)
  if (hasAnimated.current) return
  // Skip if OS or user has set reduced-motion
  if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) return

  hasAnimated.current = true

  // Query spans inside .title-heading; they are rendered synchronously above
  const spans = document.querySelectorAll('.title-heading .title-letter')
  if (spans.length === 0) return

  gsap.from(spans, {
    opacity: 0,
    y: 10,
    duration: 0.35,
    stagger: 0.05,
    ease: 'power2.out',
    clearProps: 'opacity,transform',  // clean up inline styles after animation
  })
}, [])  // empty deps — run once on mount, never re-run
```

4. Update the `<h1>` render to split "FLAPPY 3D" into individual character spans.
   Space character should also be a span (with a non-breaking space) so GSAP
   selects the right count:

```typescript
const LOGO_TEXT = 'FLAPPY 3D'
const logoLetters = LOGO_TEXT.split('')  // ['F','L','A','P','P','Y',' ','3','D']
```

Replace the current h1 render:
```typescript
// FROM:
h('h1', { className: 'title-heading' }, 'FLAPPY 3D'),

// TO:
h('h1', { className: 'title-heading' },
  ...logoLetters.map((ch, i) =>
    h('span', { key: i, className: 'title-letter' },
      ch === ' ' ? ' ' : ch,  // non-breaking space preserves visual gap
    )
  )
),
```

The `logoLetters` constant and the `hasAnimated` ref should be defined inside
the component function body (before the return / h() call).

Note: The `useEffect(() => {...}, [])` with empty deps array fires once after
first mount. Preact's `useRef` returns the same ref object across re-renders,
so `hasAnimated.current = true` persists across state changes (e.g. when
`active` prop toggles false→true on back-to-title navigation). This correctly
prevents re-animation without needing any external state.

The GSAP `from()` tween starts spans at opacity:0,y:10 and animates to their
natural CSS values (opacity:1, y:0). `clearProps: 'opacity,transform'` removes
the GSAP inline style after completion so the element returns to pure CSS
control.

(it currently fires unconditionally at 2s). Move the animation to a

```css
.title-cta {
  font-size: 1.1rem;

  from { opacity: 0.6; }
  to   { opacity: 1.0; }
}
@media (prefers-reduced-motion: reduce) {
  .title-cta.pulse {
    animation: none;
    opacity: 0.85;
  }
}
```

Note on the existing `@keyframes ctaPulse` already in the file: replace it
with the version above (keep the same name, update the animation values to use
`from`/`to` which is equivalent to the existing `0%/50%/100%` pattern but
cleaner for an `alternate` direction). The `alternate` direction means the
keyframe goes `from`→`to`→`from`→... achieving the 0.6↔1.0 cycle — the
effective period for one full 0.6→1.0→0.6 cycle is 3.2s (two 1.6s legs),
which feels right for a subtle pulse.

If `alternate` feels too slow, use `infinite` without `alternate` and keep the
`0%, 100% { opacity: 0.6; } 50% { opacity: 1.0; }` pattern with `1.6s` total.
Use this form for a guaranteed 1.6s visible cycle:
```css
.title-cta.pulse {
  animation: ctaPulse 1.6s ease-in-out infinite;
}
@keyframes ctaPulse {
  0%, 100% { opacity: 0.6; }
  50%      { opacity: 1.0; }
}
```

`@media (prefers-reduced-motion: reduce)` rule handles it as a fallback.

```typescript
const reducedMotion = window.matchMedia('(prefers-reduced-motion: reduce)').matches
```

Update the CTA element render:
```typescript
// FROM:
```

  provides a second layer of protection in case JS check is skipped or cached

No storage prop threading needed — the OS matchMedia check is sufficient for
the CTA animation gate (per D-12/D-13 resolution: CSS media query handles OS
setting; Settings toggle in storage is lower priority for this CSS effect).

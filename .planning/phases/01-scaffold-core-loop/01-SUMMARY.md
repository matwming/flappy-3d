## What Was Built

The minimum playable substrate: a hardened Three.js scene where an orange flattened-sphere "bird" falls under fixed-timestep gravity, flaps upward on Space/click/tap, and triggers a death stub when it hits a static test obstacle or the world floor/ceiling.

No game state machine yet (Phase 2). No menus, audio, particles, post-FX, or PWA yet (Phases 3–4). Just the spine — every system that follows plugs into the contracts established here.

## Phase 2 Hand-Off Notes

1. **Death stub replacement:** `CollisionSystem.die()` currently calls `console.warn(...)` + `this.loop.stop()`. Phase 2's XState machine should be injected so this becomes `this.actor.send({ type: 'HIT' })` instead. The `loop.stop()` line goes away — the machine drives the lifecycle.

2. **Bird placeholder:** Still a flattened sphere. Phase 2's VIS-01 (cel-shaded toon material) and possibly a simple modeled bird is the natural upgrade.

3. **Test obstacle:** Single static box at (2, 0, 0). Phase 2 (CORE-05/06) replaces this with a pooled ObstaclePair spawner.

4. **No camera scroll yet.** Phase 2 needs to decide camera follow strategy (fixed camera with bird scrolling vs. moving camera).

5. **Constants imported by name in physics** — when difficulty ramp lands, mutate via a config object rather than re-`import` (constants are immutable bindings).

# Phase 04 Plan 01: PWA Setup Summary

**One-liner:** vite-plugin-pwa with generateSW + autoUpdate + Workbox precache for offline play, GitHub Pages sub-path base `/flappy-3d/`, three procedural PNG icons generated via pure-Node zlib encoder.

## What Was Built

### Task 1: PWA icon generation (commit 23be80a)

A zero-dependency CommonJS script (`scripts/generate-icons.cjs`) that uses Node's built-in `zlib.deflateSync` to write valid PNG files from scratch — no external packages. Implements a minimal PNG encoder: 8-byte signature + IHDR + IDAT (zlib-deflated RGB scanlines) + IEND, with CRC32 computed via a lookup table.

Three icons generated:
- `public/icons/icon-192.png` — 192x192, sky-blue (#7ec8e3) bg + orange (#ff7043) circle radius 64
- `public/icons/icon-512.png` — 512x512, same colors, radius 170
- `public/icons/icon-maskable-512.png` — 512x512, maskable with 20% safe-zone padding (radius 140)

### Task 2: vite-plugin-pwa configuration (commit b0936cb)

- Installed `vite-plugin-pwa@1.2.0` as devDependency
- Set `base: '/flappy-3d/'` for GitHub Pages sub-path deployment
- Configured `VitePWA` with `registerType: 'autoUpdate'`, `strategies: 'generateSW'`
- Workbox: `skipWaiting: true`, `clientsClaim: true`, `globPatterns` includes `mp3`, `maximumFileSizeToCacheInBytes: 5_000_000`
- Runtime caching: `CacheFirst` for `*.mp3` (audio-cache, 1-year TTL, maxEntries 10)
- Web App Manifest inline: name="Flappy 3D", theme_color="#7ec8e3", background_color="#1a1a1a", display=standalone, start_url/scope="/flappy-3d/"
- Added `<meta name="theme-color" content="#7ec8e3">` to index.html
- Build produces `dist/sw.js`, `dist/workbox-*.js`, `dist/manifest.webmanifest`, `dist/registerSW.js`
